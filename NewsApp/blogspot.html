News App adalah aplikasi Android yang menampilkan berita terkini dari REST API (NewsAPI). Aplikasi ini dibuat dengan arsitektur MVVM dan memakai Jetpack Compose, Kotlin Coroutines, Retrofit, StateFlow, serta Coil untuk memuat gambar.

Tampilan Awal

Pada tampilan awal, aplikasi dibuat mengikuti panduan dasar pada modul. Fokusnya cuma satu: menampilkan daftar berita dari internet ke layar. Berita diambil dari endpoint top-headlines NewsAPI lewat Retrofit, lalu hasilnya dialirkan ke UI menggunakan StateFlow dari dalam ViewModel.

Di versi ini hanya ada dua halaman, yaitu Home (daftar berita) dan Detail (isi berita). Setiap berita ditampilkan dalam bentuk Card sederhana berisi gambar dan judul saja. Aplikasi juga sudah menangani tiga kondisi state: Loading (muncul lingkaran CircularProgressIndicator), Success (daftar berita tampil), dan Error (muncul pesan kesalahan beserta tombol Retry).

Secara tampilan masih sangat polos. Semua kartu berbentuk seragam, belum ada fitur pencarian, belum ada fitur simpan berita, dan belum ada navigasi tab. Tujuannya memang untuk memastikan alur data dari API sampai ke layar berjalan dulu dengan benar.


Tampilan Setelah Tambahan Fitur

Pada pembaruan kali ini, saya merapikan tampilan agar lebih modern sekaligus menambah beberapa fitur baru. Berikut perubahan yang dilakukan:

  • Navigasi Tab (Home, Search, Saved): Menambahkan tiga tab di bagian atas memakai PrimaryTabRow. Setiap tab punya ikon dan label sendiri, sehingga pengguna bisa berpindah antar halaman dengan mudah.
  • Perombakan Halaman Home: Berita pertama kini tampil sebagai featured card berukuran besar dengan judul di atas gambar dan efek gradient gelap agar teks mudah dibaca. Di bawahnya ada bagian "Breaking News" berisi daftar berita dalam bentuk baris ringkas (gambar kecil di kiri, judul beserta kategori dan waktu di kanan, misalnya "TECHNOLOGY · 2 hours ago").
  • Fitur Search: Menambahkan halaman pencarian yang memakai endpoint everything dari NewsAPI. Pengguna tinggal mengetik kata kunci, lalu hasilnya muncul dalam daftar "Search Result". Halaman ini juga punya state kosong, loading, dan error sendiri.
  • Fitur Save dengan Room Database: Berita bisa disimpan secara lokal di perangkat menggunakan Room Database. Berita yang disimpan langsung muncul di tab Saved secara real-time berkat penggunaan Flow. Data tetap tersimpan walaupun aplikasi ditutup.
  • Halaman Detail yang Lebih Lengkap: Halaman Detail kini menampilkan kategori, waktu terbit, dan penulis. Di bagian bawah ada tombol bookmark untuk menyimpan/menghapus berita (ikon berubah sesuai status simpan) dan tombol share untuk membagikan berita ke aplikasi lain lewat Intent.
  • Pembaruan Tampilan: Mengganti seluruh ikon dari emoji menjadi Material Icons resmi, serta menyeragamkan warna aplikasi ke warna biru sebagai identitas (brand color) agar terlihat lebih rapi dan konsisten.

Hal yang Dipelajari

Melalui proses pembuatan dan pengembangan aplikasi News App ini, ada beberapa hal penting yang saya pelajari:

  • Konsumsi REST API: Cara mengambil data dari web service (NewsAPI) memakai Retrofit dan Kotlin Coroutines, lalu mengubah respons JSON menjadi objek Kotlin dengan bantuan Gson.
  • Arsitektur MVVM dan State Management: Memisahkan kode menjadi beberapa lapisan (data, repository, viewmodel, ui) dan mengelola kondisi tampilan dengan StateFlow melalui sealed class (Loading, Success, Error) agar UI mudah merespons perubahan data.
  • Penyimpanan Lokal dengan Room: Membuat Entity, DAO, dan database untuk menyimpan berita favorit, serta memakai Flow supaya daftar tersimpan otomatis ter-update tanpa perlu di-refresh manual.
  • Navigation Compose: Mengatur perpindahan antar halaman (Home ke Detail) dan mengirim data berita yang dipilih lewat ViewModel yang dipakai bersama.
  • Memuat Gambar dari Internet: Menggunakan Coil (AsyncImage) untuk menampilkan gambar berita langsung dari URL.
  • Troubleshooting Kompatibilitas Versi: Menyesuaikan versi library dengan Kotlin 2.2.10 dan AGP 9, termasuk mengatasi konflik antara built-in Kotlin AGP dengan Kotlin Symbol Processing (KSP) saat memasang Room (menambahkan android.disallowKotlinSourceSets=false), serta memilih versi KSP dan Room yang cocok agar aplikasi bisa build dengan lancar.
