Tampilan Awal

Pada tampilan awal, aplikasi Registrasi Siswa memiliki desain yang sangat mendasar. Form input menggunakan TextField standar tanpa garis luar yang tegas. Tombol tambah siswa terlihat biasa, dan daftar siswa yang ditampilkan belum sepenuhnya terlihat rapi karena kurangnya penggunaan batas card dan padding shadow yang mempertegas elemen daftar. Selain itu, fitur operasi seperti edit dan delete juga belum memunculkan interaksi konfirmasi atau popup yang ramah pengguna.


Tampilan Setelah Tambahan Fitur

Pada iterasi pembaruan proyek kali ini, saya telah melakukan beberapa peningkatan baik dari segi fungsionalitas (UX) maupun antarmuka (UI). Berikut adalah fitur-fitur yang diubah dan ditambahkan ke dalam aplikasi:

  • Pembaruan Desain Input Form dan Card List: Mengganti komponen input dari TextField biasa menjadi OutlinedTextField berbentuk kapsul (rounded corners 12.dp) agar terlihat lebih modern. Bagian daftar siswa juga diubah ke dalam bentuk Card dengan warna dasar putih dan elevasi bayangan, lengkap dengan pembaruan warna ikon taksi (ungu untuk Edit, merah untuk Delete).
  • Avatar Profil dengan Warna Acak Konsisten: Menambahkan inisial nama membulat (maksimal 2 huruf) sebagai avatar profil. Warnanya dipilih secara acak dari palet warna pastel, namun tetap konsisten menggunakan algoritma operasi hash dari string email pengguna sehingga tidak berubah-ubah tiap direload.
  • Fitur Edit Menggunakan Popup Interaktif: Memisahkan fungsionalitas Edit Data menjadi form tersendiri layaknya pop-up. Memanfaatkan komponen AlertDialog, saat tombol Edit ditekan, sebuah dialog mandiri muncul tepat di tengah layar yang memungkinkan input pengubahan nama dan email secara independen tanpa mengganggu form tambah data.
  • Popup Konfirmasi Hapus Data: Mencegah penghapusan data secara tidak sengaja oleh pengguna dengan menambahkan sebuah AlertDialog konfirmasi. Saat ikon tempat sampah ditekan, akan muncul popup pertanyaan, dan sistem baru akan menghapus data tersebut dari Room Database jika tombol konfirmasi ditekan.

Hal yang Dipelajari

Melalui proses perbaikan dan pengembangan fitur aplikasi Registrasi Siswa ini, saya telah mempraktikkan dan mempelajari sejumlah hal signifikan:

  • Troubleshooting Build Configuration Android: Mengatasi berbagai persoalan pada konfigurasi Gradle secara langsung, seperti inkonsistensi versi plugin Jetpack Compose (Kotlin 2.0+), perbaikan missing API level, penambahan android.useAndroidX=true, serta meratakan konfigurasi JVM Target versi 11 antara Javac dan Kotlin Symbol Processing (KSP).
  • State Management dan Dialog di Compose: Memahami secara nyata mekanisme *trigger* untuk menyembunyikan dan memunculkan pop-up (AlertDialog) menggunakan bantuan state hoisting (remember { mutableStateOf() }).
  • Manipulasi String & Hash: Menggunakan operasi dasar bawaan Kotlin untuk mengekstrak string nama menjadi inisial ganda (take(2), uppercase()) serta cara cemerlang memanipulasi perhitungan hash algorithm (abs(string.hashCode() % warna.size)) demi membuat warna avatar unik namun konsisten bagi setiap subjek data.
  • Peningkatan Pola Desain (Material3): Praktik langsung dalam merapikan hierarki layout menjadi komponen UI standar Material3 (OutlinedTextField, Card Elevasi) serta pemakaian Modifier dan pembagian tata letak Row/Column di Jetpack Compose.
